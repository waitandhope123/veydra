# Contributing to Veydra

Thank you for your interest in contributing to **Veydra**, an open, EV-native vehicle design project.  
This repository aims to explore and document a novel electric vehicle architecture focused on safety, modularity, efficiency, and clean-sheet engineering.  
Contributions of all kinds are welcome ‚Äî from technical research to visual design to documentation.

---

# üìò 1. How to Contribute

There are several ways to participate in the project:

### **A. Documentation Contributions**
- Corrections, expansions, or clarifications to any file in `/docs`
- Additional research on EV technologies, materials, safety systems, or aerodynamics
- Visual diagrams or charts to enhance understanding

### **B. Concept Artwork or Renders**
- Exterior design sketches  
- Module concepts (pods, interiors, roof systems)  
- Aerodynamic studies  
- Technical diagrams  

### **C. Engineering Research**
- Aerodynamic analysis (CFD or theoretical)  
- Structural design proposals  
- Safety or crash modeling references  
- Powertrain analysis  
- Battery chemistry and energy flow models  

### **D. Software & Control System Concepts**
- Predictive energy management frameworks  
- Defensive Mode logic flows  
- User interface mockups  
- Mode control strategies  

### **E. Project Structure**
- Repository improvements  
- Better organization or naming  
- Quality-of-life changes to documentation

# üõ†Ô∏è 2. Getting Started

### **1. Fork the repository**
Click the **Fork** button at the top right of the GitHub page.

### **2. Clone your fork**
```bash
git clone https://github.com/<your-username>/veydra.git
```

### **3. Create a branch**
```bash
git checkout -b feature/my-new-idea
```

### **4. Make your changes**
Follow existing conventions for file names, formatting, and folder structure.

### **5. Commit with a descriptive message**
```bash
git commit -m "Add analysis of rear pod aerodynamics"
```

### **6. Push**
```bash
git push origin feature/my-new-idea
```

### **7. Create a Pull Request**
Open a PR and describe what you added and why.

# üìê 3. Style Guidelines

### **Documentation Style**
- Use clear, technical language when appropriate  
- Prefer Markdown formatting  
- Keep sections well-organized  
- Avoid overclaiming; keep analysis grounded in known physics  

### **Naming Conventions**
- Filenames: lowercase, hyphens or underscores  
- Directories: lowercase  
- Images: descriptive, lowercase filenames  

### **Technical Accuracy**
Contributors should cite reputable sources or state when something is theoretical or proposed.
